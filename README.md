# 🚀 小满 CRM 询价附件自动同步系统

## 📋 项目概述
本项目是一套基于 **Python + Playwright** 开发的工业级 RPA（机器人流程自动化）系统。旨在解决小满（OKKI）CRM 系统中，询价历史附件与产品基本信息附件“数据孤岛”的问题。系统能够自动执行从产品搜索、询价判定、附件提取、到原位回传的全链路闭环操作。

## 🌟 核心能力
1.  **智能批处理**：支持通过 `tasks.txt` 批量导入产品 ID，一键启动，无人值守运行。
2.  **深度穿透下载**：采用“新标签页拦截”技术，精准定位询价任务中的 Excel 报价文件，并保持原始文件名下载。
3.  **静默后台运行**：支持 **Headless（无头）** 模式，通过 1080P 虚拟视口仿真，在不干扰用户正常办公（鼠标、键盘操作）的情况下，在后台静默执行任务。
4.  **容灾登录哨兵**：内置登录状态检测机制。当 Cookie 过期时自动暂停并提醒用户手动补登，确保长效任务的稳定性。
5.  **自动化巡检报告**：任务完成后自动生成 `.csv` 格式的执行报告，记录每个 ID 的处理状态（成功、查无产品、无询价历史等）。

## 🛠 技术架构
* **自动化引擎**：Playwright (Chromium/Edge 内核)
* **执行模式**：异步 IO (Asyncio)
* **模拟策略**：物理级鼠标事件触发 (Avoid Event Bubbling) + 虚拟 1080P Viewport
* **文件系统**：动态目录管理 (Downloads/Reports 分离)

## 🔄 业务流转逻辑
1.  **初始化**：读取 `tasks.txt`，检查 Edge 自动化环境与登录状态。
2.  **精准搜索**：利用 `product-name-link` 类名锚点定位目标产品。
3.  **询价探测**：跳转至“询价历史”标签页，通过空状态元素（Empty State）智能判定是否有历史记录。
4.  **穿透跳转**：锁定 `jump-link` 任务编号，拦截新打开的任务详情标签页。
5.  **附件提取**：扫描详情页中的“下载”按钮，判断，过滤出 `.xls/.xlsx` 文件并接管下载流。
6.  **路由回传**：瞬移至该产品的“附件”标签页，拦截 Windows 文件选择器，静默注入本地文件完成回传。
7.  **归档报告**：记录结果并处理下一个 ID。

## 📂 项目结构
```text
Test-CRM/
├── login_check.py       # 核心程序代码
├── tasks.txt            # 待处理产品编号清单 (一行一个)
├── downloads/           # 下载附件暂存区 (保持原名)
├── reports/             # 自动化执行报告 (.csv)
└── logs/                # (可选) 调试截图与错误追踪
```

## 🚀 使用指南
1.  **环境准备**：确保安装了 Python 3.9+ 及 Playwright 环境。
2.  **准备清单**：在 `tasks.txt` 中填入产品 ID。
3.  **启动系统**：执行命令 `python login_check.py`。
4.  **查看结果**：任务结束后，在 `reports/` 目录下查看当天的执行摘要。

## ⚠️ 注意事项
* **视口设置**：无头模式下必须强制开启 1920x1080 视口，否则 CRM 响应式布局会导致元素隐藏。
* **登录哨兵**：检测登录态并支持人工介入。
